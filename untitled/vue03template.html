템플릿 문법

Vue.js는 렌더링된 돔을 기본 Vue인스턴스의 데이터에 선언적으로 바인딩 할 수 잇는 HTML 기반 템플릿 구문을 사용한다. 모든 Vue.js 템플릿은 스펙을 호환하는 브라우저 및 HTML파서로 구문 분석할 수 있는 유효한 HTML이다.

내부적으로 Vue는 템플릿을 가상 DOM렌더링 함수로 컴파일 한다. 반응형 시스템과 결합되 vue는 앱 상태가 변겯될때 최소한으로 DOM을 조작하고 다시 적용할 수 있는 최소한의 컴포넌트를 지능적으로 파악할 수 있다.

가상 Dom개념에 익숙하고 자바스크립트의 기본기능을 선호하는 경우 템플릿 대신 렌더링 함수를 직접 작성할 수 있으며 선택사항으로 JSX를 지원한다.

보간법(Interpolation)

문자열

데이터 바인딩의 가장 기본형태는 "Mustache"구문 (이중 중괄호)를 이용한 텍스트 보간

Mustache 태그는 해당 데이터 객체의 msg 속성값으로 대체된다. 또한 데이터 객체의 msg속성이 변경될 때마다 갱신된다.

v-once 디렉티브를 사용하여 데이터 변경 시 업데이트 되징 낳는 일회성 보간을 수행할 수 있지만, 같은 노드의 바인딩에도 영향을 미친다

원시 HTML

mustaches는 HTML이 아닌 일반 텍스트로 데이터를 해석한다. 실제 HTML을 출력하려면 v-html 디렉티브를 사용해야 한다.

속성

Mustaches는 HTML속성에서 사용할 수 없다. 대신 v-vind디렉티브를 사용하라

자바스크립트 표현식

mustache는 구문이 아닌 표현식만이 작동된다.

디렉티브

디렉티브는 v-접두사가 있는 특수 속성이다. 디텍티브 속성 값은 단일 자바스크립트표현식이 된다. 디렉티브의 역할은 표현식의 값이 변경될 대 사이드 이펙트를 반응적으로 돔에 적용하는 것이다.

전달인자

일부 디렉티브는 콜론으로 표시되는 전달인자를 사용할 수 있다 예를 들어 v-bind 디렉티브는 반응적으로 HTML속성을 갱신하는데 사용된다.

여기서 href는 전달인자로 엘리먼트의 href속성을 표현식 url의 값에 바인드하는 v-bind 디렉티브에게 알려준다.

또 다른 예로 돔 이벤트를 수신하는 v-on디렉티브가 있다.
